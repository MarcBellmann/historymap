# HistoryMap – Claude Instructions

## Package Manager

Use **Bun** for all package management and script execution:

- `bun install` instead of npm/yarn/pnpm
- `bun run dev` to start the dev server
- `bunx <package>` instead of npx
- `bunx tsc --noEmit` to type-check

## Node.js Requirement

Vite 7 requires Node.js ≥ 22. Always run commands with Node 22 in PATH:

```sh
PATH="/home/marc/.nvm/versions/node/v22.20.0/bin:/home/marc/.bun/bin:$PATH" bun run dev
```

The dev server writes to `/tmp/historymap-dev.log` and `/tmp/historymap-dev.pid`.

## Tech Stack

- **Framework**: TanStack Start (React 19, TypeScript) — SSR via Vite Environments API
- **Map**: MapLibre GL JS via `react-map-gl` v8
- **Styling**: Tailwind CSS v4 + Radix UI primitives
- **i18n**: react-i18next, resources bundled inline (DE/EN)
- **Data**: Flat JSON/GeoJSON files in `app/data/`

## Project Structure

```
app/
  routes/         # TanStack Router file-based routes
  components/

    Timeline/     # TimelineSlider (prev/next navigation)

  data/

  lib/            # i18n, dataUtils, timeUtils, cn
  types/          # history.ts type definitions
public/

  ne_110m_ocean.geojson # Bundled Natural Earth data
  ne_110m_rivers.geojson
  ne_110m_lakes.geojson
```

## Key Architecture Notes

- `app/router.tsx` must export `createRouter`, `getRouter`, `startInstance`
- `app/routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin` — do not edit
- GeoJSON imports work via custom `geojsonPlugin()` in `vite.config.ts`
- `@vitejs/plugin-react` must be in the Vite plugins list for JSX to work
- Map click events are handled at `<Map>` level via `interactiveLayerIds` — not on individual `<Layer>` components
- Year encoding: negative integers = BC (e.g. `-44` = 44 BC), positive = AD

## Data Model

- `cities.json`: `{ id, coordinates, startYear, endYear, importance (1-3), culturalSphere[], labels, description }`
- `regions.geojson`: GeoJSON FeatureCollection, properties: `{ id, startYear, endYear, culturalSphere, color, opacity, labels, description }`
- `events.json`: `{ id, coordinates, year, type, culturalSphere[], labels, description }`
- Filter logic: `startYear <= currentYear && (endYear === null || endYear >= currentYear)`
